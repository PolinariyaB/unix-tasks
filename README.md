# UNIX Tasks

## Task 1 — Sparse File Copier

Программа на C для копирования файлов с созданием разреженных (sparse) файлов.  
Нулевые блоки не записываются на диск, а пропускаются через `lseek()`, что экономит место.

### Что делает

- Читает данные из файла или stdin
- Разбивает на блоки (размер настраивается)
- Блоки из одних нулей пропускает (создаёт "дыры")
- Блоки с данными записывает обычным `write()`
- Восстанавливает точный размер через `ftruncate()`

### Системные вызовы

- `open()` / `read()` / `write()`
- `lseek()` — для пропуска нулевых блоков
- `ftruncate()` — для установки размера
- `close()`

### Сборка

```bash
cd task1-sparse
make
./runme.sh
```


## Task 2 — File Lock

Программа реализует примитивную файловую блокировку через lock-файл <myfile>.lck.

### Сборка

```bash
cd task2-lock
make
```

### Запуск

```bash
./lockprog [-s stats.txt] [-d delay_ms] myfile
```

### Параметры

- `myfile` — базовый путь; lock-файл: `myfile.lck`
- `-s` — путь к файлу статистики (по умолчанию `stats.txt`)
- `-d` — пауза между попытками захвата lock (по умолчанию `100`)

### Скрипт:

- собирает проект
- запускает 10 параллельных процессов на shared.txt
- через TEST_DURATION секунд отправляет всем SIGINT
- дожидается завершения
- формирует отчёт result.txt

### Очистка

```bash
cd task2-lock
make clean
rm -f result.txt stats.txt shared.txt shared.txt.lck
```


## Task 3 myinit

`myinit` читает конфигурацию (абсолютные пути), демонизируется, запускает дочерние процессы и следит за ними.

* при завершении процесса — перезапускает его
* по `SIGHUP` — перечитывает конфиг и перезапускает всех
* лог пишет в `/tmp/myinit.log`

Для тестов PID можно сохранить через переменную `MYINIT_PIDFILE`.

## Сборка

```bash
cd task3-myinit
make
```

## Запуск

```bash
./myinit /absolute/path/to/config.txt
```

## Формат конфига

```
/path/to/program arg1 arg2 ... /abs/stdin /abs/stdout
```

* последние 2 поля — stdin/stdout
* всё остальное — аргументы для `execv`
* все пути должны быть абсолютные

## Тестирование

```bash
bash runme.sh
```

Скрипт:

* запускает 3 процесса
* проверяет их наличие
* убивает один → проверяет рестарт
* шлёт `SIGHUP` → остаётся 1 процесс
* проверяет лог и пишет `result.txt`

## Очистка

```bash
make clean
```

## Task 4 brownian-bot

Реализация клиент-серверной системы с UNIX-сокетами и асинхронным вводом-выводом (poll).

## Состав

server

- Читает имя сокета из config, создаёт /tmp/<name>
- Однопоточный, обслуживает много клиентов через poll
- Хранит целое состояние (начинается с 0)
- Принимает числа строками, прибавляет к состоянию, отправляет результат
- Логирует в /tmp/brownian-bot.log

client

- Подключается к серверу
- Читает числа из stdin
- Отправляет строку → получает ответ → выводит в stdout

test_client

- Читает файл с числами побайтно
- Отправляет кусками (1–255 байт) с задержками
- Имитирует “медленных” клиентов
- Пишет статистику в лог

## Сборка

```bash
make clean && make
```

## Запуск

```bash
bash runme.sh
```
